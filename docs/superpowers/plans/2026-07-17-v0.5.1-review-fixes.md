> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the eight P1/P2 defects from the external v0.5.0 review so the shipped behavior matches the documented fail-closed / fresh-reallocation contracts.

**Architecture:** All fixes are local hardening of existing modules — no new subsystems. The largest change removes the ledger auto-heal (move-aside + empty rebuild) from `store::load` and gates it behind a new explicit `portool doctor --repair`; everything else is validation, ordering, or classification-order corrections.

## Global Constraints

- Exit codes are contractual: General=1, PoolExhausted=3, LockTimeout=4, usage=64, exec 126/127. New failures use `Error::General` (exit 1) unless stated.
- The post-checkout hook must never fail `git checkout` — already guaranteed by the `|| …` hook forms in `src/cmd/init.rs`; do not change hook content.
- Ledger writes stay atomic (temp file + rename); never save a registry that fails `registry.validate()`.

## Edge cases and cautions (pre-implementation checklist)

1. **Hook safety with fail-closed sync (Task 7):** the installed hook forms already guard `portool sync --quiet` with `|| …` (src/cmd/init.rs:25,33), so a corrupt ledger cannot fail `git checkout`. Do not weaken this.
2. **`doctor_rebuilds_a_lost_entry_from_the_env_file`** deletes the ledger (missing = empty, not corrupt) — must keep passing without `--repair`.
3. **Reallocate contract change (Task 4):** a pool with room only for the current block now exits 3 instead of silently keeping the block. Intended; called out in CHANGELOG.
4. **Absolute local-scope hooksPath on git < 2.26 (Task 6):** now refused (fail-closed, scope "unknown"); init prints the manual line — degraded but safe.
5. **Fast path cost (Task 3):** adds one config read+parse per sync; acceptable, and the README perf claim is softened in Task 9.
6. **`release` with `.env.portool` already absent (Task 5):** NotFound is ignored; the ledger entry is still removed (unchanged semantics).
7. **Sequencing:** Task 7 must precede Task 8 (`move_aside`/`require_usable`). Tasks 2/3/4 all edit `src/cmd/sync.rs` — execute in plan order to avoid conflicting context.
8. **Public API breakage (`store::load`, `Manifest::block_size`, `cmd::doctor::run`):** the crate is 0.x and CLI-first; acceptable, noted in CHANGELOG. Shrinking the public surface overall is a v0.6 item, out of scope here.
9. **Deferred to v0.6 (per the review's own priority list):** two-phase ledger/env updates with generations, fast-path snapshot re-validation, `LoadResult` → enum, core-crate split, benchmark-backed perf claims, fault-injection test abstraction.
