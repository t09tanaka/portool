# portool v0.10.0 P0 Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the seven P0 findings from the fifth external review (symlink-safe hook writes, a stale-backup recovery contract, a transactional `deinit`/`unhook`, machine-readable pending-move state, three-valued identity checks, strict shebang parsing) and ship as v0.10.0 with an independent review gate.

**Architecture:** Each P0 is a self-contained, independently testable change against the binary-only `portool` crate (no lib target since v0.9.0). The load-bearing new primitive is `store::open_repo_relative` — a `lstat`-per-component, fail-closed, fd-relative path opener that every hook write routes through, replacing the current `atomic_write`/`fs::create_dir_all` calls in `cmd/init.rs`. Everything else builds on existing schema-v3 ledger machinery: P0-2 adds a monotonic ledger `sequence` mirrored into `.env.portool`; P0-4 surfaces `pending_block` + a derived `state` in `ls --json`; P0-5 makes env-identity parsing three-valued.

**Tech Stack:** Rust 2021, MSRV 1.85, `clap` 4, `serde`/`serde_json`, `sha2`, `tempfile`, `nix`-free (use `std::os::unix` + `libc` only if already present — prefer `std::os::fd`/`std::fs` where possible). Unix-only (`std::os::unix`).

## Global Constraints

- **MSRV 1.85** — no API newer than Rust 1.85. `std::os::unix::fs::OpenOptionsExt`, `std::os::fd`, `openat`-style flows via `std::fs::File` + `cap_std`-free hand-rolled code are all fine on 1.85.
- **Binary-only crate** — no `pub` API contract; all modules are private. Tests are black-box CLI tests in `tests/*.rs` plus in-module `#[cfg(test)]` unit tests. There is no `lib.rs`.
- **Unix-only** — `std::os::unix` freely usable; no Windows path handling.
- **Ledger schema is versioned** — any on-disk ledger change bumps `Registry::CURRENT_VERSION` and adds a migration arm in `registry::from_json`, keeping every worktree `block` verbatim.
- **`.env.portool` is `source`d by users** — every interpolated value stays on a single inert comment line (`envfile::sanitize_comment`); never emit an executable line from untrusted input.
- **Hooks always exit 0** — a portool failure must never fail `git checkout`/`git worktree add`.
- **Test port-range isolation** — any test that binds ports or exact-asserts block values must pin its own `range = [X, Y]` in a written config. Used ranges (do not reuse): 3000s/3800s/3900s/4200-4319, 17300-17699, 18000-18299, 18500-18599, 18600-18609, 18700-18799, 18800-18809, 18900-18959, 19100-19399, 19400-19409, 19410-19444. **New ranges claimed by this plan: 19500-19559 (P0-2 recovery), 19560-19599 (P0-3 deinit), 19600-19649 (P0-4 pending-move JSON).**
- **Output sanitation** — all terminal-facing paths/text go through `display::path`/`display::text`; JSON output is unsanitized (machine interface).
- **Every command's exit code is semantic** — clap usage errors exit 64; keep portool's own codes distinct (`error::Error::exit_code`).

Add the helper (replaces reliance on `lexical_normalize` alone; keep `lexical_normalize` for the tail):

```rust
/// Canonicalizes the longest existing prefix of `path` (which resolves every
/// symlink in those components), then re-appends the remaining non-existent
/// components lexically. Unlike a bare `canonicalize`, this never fails on a
/// not-yet-created leaf; unlike a bare lexical normalize, it never leaves an
/// existing intermediate symlink unresolved.
fn canonicalize_existing_ancestor(path: &Path) -> PathBuf {
    let mut ancestor = path.to_path_buf();
    let mut tail: Vec<std::ffi::OsString> = Vec::new();
    loop {
        match ancestor.canonicalize() {
            Ok(canon) => {
                let mut out = canon;
                for part in tail.iter().rev() {
                    out.push(part);
                }
                return lexical_normalize(&out);
            }
            Err(_) => match ancestor.file_name() {
                Some(name) => {
                    tail.push(name.to_os_string());
                    if !ancestor.pop() {
                        return lexical_normalize(path);
                    }
                }
                None => return lexical_normalize(path),
            },
        }
    }
}
```

Add the safe-opener block (self-contained; uses only `std::fs`, `std::os::unix`, and `libc` — confirm `libc` is a dependency; if not, add `libc = "0.2"` to `Cargo.toml` — it is already transitively present via `tempfile`, but declare it directly):

```rust
/// A file opened under a repository boundary with every path component
/// verified non-symlink, plus its fully-resolved final path.
pub struct OpenedFile {
    pub file: fs::File,
    pub path: PathBuf,
}

/// Opens `target` for writing a fresh file, but only if every component of
/// `target` **below `boundary`** is a real directory (no symlink, no `..`,
/// nothing escaping `boundary`). The directory walk is fd-relative
/// (`openat` with `O_NOFOLLOW | O_DIRECTORY`), so no attacker can swap a
/// checked component for a symlink between the check and the open
/// (TOCTOU-safe). `boundary` must already be canonical (it is: `GitCtx`
/// realpaths both roots). Creating missing intermediate directories is the
/// caller's job via [`create_dirs_repo_relative`]; this call fails if a
/// parent is missing.
///
/// Returns the opened parent-directory fd and the leaf name so the caller
/// can `renameat`/`openat` the final file relative to it.
pub fn open_parent_dir_fd(boundary: &Path, target: &Path) -> Result<(fs::File, std::ffi::OsString)> {
    let rel = target.strip_prefix(boundary).map_err(|_| {
        Error::General(format!(
            "refusing to write {}: it is not under {}",
            target.display(),
            boundary.display()
        ))
    })?;
    let mut components: Vec<&std::ffi::OsStr> = Vec::new();
    for c in rel.components() {
        match c {
            std::path::Component::Normal(name) => components.push(name),
            // Any `..`, `.`, root, or prefix in the *relative* remainder is
            // an escape attempt; refuse.
            _ => {
                return Err(Error::General(format!(
                    "refusing to write {}: path component escapes {}",
                    target.display(),
                    boundary.display()
                )))
            }
        }
    }
    let leaf = components.pop().ok_or_else(|| {
        Error::General(format!("refusing to write {}: empty path", target.display()))
    })?;

    // Open the boundary itself with O_NOFOLLOW|O_DIRECTORY, then walk each
    // intermediate directory the same way, fd-relative.
    let mut dir = open_dir_nofollow(boundary, None)?;
    for name in components {
        dir = open_dir_nofollow(Path::new(name), Some(dir.as_raw_fd())).map_err(|e| {
            Error::General(format!(
                "refusing to write under {}: intermediate '{}' is not a real directory ({e})",
                boundary.display(),
                Path::new(name).display()
            ))
        })?;
    }
    Ok((dir, leaf.to_os_string()))
}

/// `openat(dirfd, name, O_RDONLY|O_NOFOLLOW|O_DIRECTORY)` — or the boundary
/// itself when `dirfd` is `None`. Fails on a symlink (`ELOOP`) or non-dir.
fn open_dir_nofollow(name: &Path, dirfd: Option<std::os::unix::io::RawFd>) -> std::io::Result<fs::File> {
    use std::ffi::CString;
    use std::os::unix::ffi::OsStrExt;
    let c = CString::new(name.as_os_str().as_bytes())
        .map_err(|_| std::io::Error::from(std::io::ErrorKind::InvalidInput))?;
    let flags = libc::O_RDONLY | libc::O_NOFOLLOW | libc::O_DIRECTORY | libc::O_CLOEXEC;
    let fd = match dirfd {
        Some(d) => unsafe { libc::openat(d, c.as_ptr(), flags) },
        None => unsafe { libc::open(c.as_ptr(), flags) },
    };
    if fd < 0 {
        return Err(std::io::Error::last_os_error());
    }
    // SAFETY: fd is a freshly opened, owned descriptor.
    Ok(unsafe { <fs::File as std::os::unix::io::FromRawFd>::from_raw_fd(fd) })
}

/// Atomically writes `contents` to `target` with `mode`, refusing if any
/// component below `boundary` is a symlink or escapes it. Temp file +
/// `renameat`, both relative to the verified parent-dir fd. Returns the
/// final path actually written.
pub fn atomic_write_within(
    boundary: &Path,
    target: &Path,
    contents: &[u8],
    mode: u32,
) -> Result<PathBuf> {
    use std::ffi::CString;
    use std::io::Write;
    use std::os::unix::ffi::OsStrExt;

    let (dir, leaf) = open_parent_dir_fd(boundary, target)?;
    let dirfd = dir.as_raw_fd();

    // Create the temp file O_CREAT|O_EXCL|O_NOFOLLOW relative to dirfd.
    let tmp_name = format!(".portool-tmp-{}-{}", std::process::id(), leaf.to_string_lossy());
    let tmp_c = CString::new(tmp_name.as_bytes())
        .map_err(|_| Error::General("temp name contains NUL".into()))?;
    let flags = libc::O_WRONLY | libc::O_CREAT | libc::O_EXCL | libc::O_NOFOLLOW | libc::O_CLOEXEC;
    let fd = unsafe { libc::openat(dirfd, tmp_c.as_ptr(), flags, mode as libc::c_uint) };
    if fd < 0 {
        return Err(Error::from(std::io::Error::last_os_error()));
    }
    let mut tmp = unsafe { <fs::File as std::os::unix::io::FromRawFd>::from_raw_fd(fd) };
    let write_result = (|| -> std::io::Result<()> {
        tmp.write_all(contents)?;
        tmp.sync_all()?;
        Ok(())
    })();
    if let Err(e) = write_result {
        unsafe { libc::unlinkat(dirfd, tmp_c.as_ptr(), 0) };
        return Err(Error::from(e));
    }
    drop(tmp);
    let leaf_c = CString::new(leaf.as_bytes())
        .map_err(|_| Error::General("leaf name contains NUL".into()))?;
    let rc = unsafe { libc::renameat(dirfd, tmp_c.as_ptr(), dirfd, leaf_c.as_ptr()) };
    if rc < 0 {
        let e = std::io::Error::last_os_error();
        unsafe { libc::unlinkat(dirfd, tmp_c.as_ptr(), 0) };
        return Err(Error::from(e));
    }
    // Re-derive the final path for the caller/report.
    Ok(boundary.join(target.strip_prefix(boundary).unwrap()))
}

/// Ensures every intermediate directory of `target` exists under `boundary`,
/// creating missing ones with `mkdirat` (mode 0755) — but only when the
/// existing prefix is symlink-free. Refuses to `mkdir` through a symlink.
pub fn create_dirs_repo_relative(boundary: &Path, target: &Path) -> Result<()> {
    use std::ffi::CString;
    use std::os::unix::ffi::OsStrExt;
    let rel = target.strip_prefix(boundary).map_err(|_| {
        Error::General(format!("{} is not under {}", target.display(), boundary.display()))
    })?;
    let mut names: Vec<&std::ffi::OsStr> = Vec::new();
    for c in rel.components() {
        match c {
            std::path::Component::Normal(name) => names.push(name),
            _ => return Err(Error::General(format!(
                "{} has a component escaping {}", target.display(), boundary.display()
            ))),
        }
    }
    names.pop(); // leaf is a file, not a dir
    let mut dir = open_dir_nofollow(boundary, None)?;
    for name in names {
        let c = CString::new(name.as_bytes())
            .map_err(|_| Error::General("dir name contains NUL".into()))?;
        // Try to open it (must be a real dir); if missing, mkdirat then open.
        match open_dir_nofollow(Path::new(name), Some(dir.as_raw_fd())) {
            Ok(next) => dir = next,
            Err(e) if e.kind() == std::io::ErrorKind::NotFound => {
                let rc = unsafe { libc::mkdirat(dir.as_raw_fd(), c.as_ptr(), 0o755) };
                if rc < 0 {
                    return Err(Error::from(std::io::Error::last_os_error()));
                }
                dir = open_dir_nofollow(Path::new(name), Some(dir.as_raw_fd()))
                    .map_err(Error::from)?;
            }
            Err(e) => return Err(Error::General(format!(
                "refusing to create dirs under {}: '{}' is not a real directory ({e})",
                boundary.display(), Path::new(name).display()
            ))),
        }
    }
    Ok(())
}
```

```rust
/// The repository boundary a given hooks location's files must stay within:
/// the worktree root for Husky (`.husky/...`), the common dir for git's
/// default/custom hooks dir. Both are already canonical (GitCtx realpaths).
fn hook_boundary(loc: &HooksLocation, ctx: &GitCtx) -> Option<PathBuf> {
    match loc {
        HooksLocation::Husky { .. } => Some(ctx.worktree_root.clone()),
        HooksLocation::GitDefault { hooks_dir } | HooksLocation::Custom { hooks_dir } => {
            // The hooks dir lives under common_dir (default) or the worktree
            // (a relative custom path). Use whichever canonical root is an
            // ancestor; if neither is, refuse.
            if hooks_dir.starts_with(&ctx.common_dir) {
                Some(ctx.common_dir.clone())
            } else if hooks_dir.starts_with(&ctx.worktree_root) {
                Some(ctx.worktree_root.clone())
            } else {
                None
            }
        }
        HooksLocation::Missing { .. } | HooksLocation::SharedScope { .. } => None,
    }
}
```

Rewrite `atomic_write` in `init.rs` to delegate (keeping the symlink pre-check that already exists in `install_into`, now backed by the fd-relative guarantee):

```rust
/// Writes `contents` to `path` atomically and symlink-safely: the write is
/// refused if any component of `path` below `boundary` is a symlink or
/// escapes it (external review v0.10 P0-1). Preserves nothing about an
/// existing file's mode — callers pass the intended `mode`.
fn atomic_write(boundary: &Path, path: &Path, contents: &[u8], mode: u32) -> Result<()> {
    crate::store::create_dirs_repo_relative(boundary, path)?;
    crate::store::atomic_write_within(boundary, path, contents, mode)?;
    Ok(())
}
```

Every `atomic_write(hook_path, bytes)` call in `install_into`/`deinit_hook` becomes `atomic_write(boundary, hook_path, bytes, mode)`, and `set_mode` is folded into the write (pass the intended mode; drop separate `set_mode`/`ensure_executable` chmod calls on the rewrite path since the mode is set atomically at create). For the **in-place refresh of an existing** hook (owned standalone / managed block), read stays via `fs::read_to_string` (fine — reading a symlink target is not a write escape, and the symlink case is already rejected before any read by the existing `symlink_metadata` check at the top of `install_into`). The *write-back* uses the boundary-safe path.

`install_hook` re-validates the boundary right before writing: after `HooksLocation::resolve`, recompute `hook_boundary`; if it is `None` for a variant that should have one, treat as `ManualRequired`/refuse.

```rust
pub fn deinit(keep_allocations: bool) -> Result<()> {
    use crate::{lock, paths, store};
    use std::time::Duration;

    let cwd = std::env::current_dir()?;
    let ctx = GitCtx::discover(&cwd)?;
    let loc = HooksLocation::resolve(&ctx);
    let common_dir_key = ctx.common_dir.to_string_lossy().into_owned();

    // 1. Stop/remove hooks first, so a concurrent checkout can't re-create
    //    the env we're about to delete.
    let hook_results = remove_hooks(&ctx)?;
    // 2. Verify each hook is actually neutralized.
    let mut residue: Vec<String> = Vec::new();
    for name in ["post-checkout", "post-merge"] {
        if !hook_is_neutralized(&loc, name) {
            residue.push(format!("hook:{name}"));
        }
    }

    if !keep_allocations {
        let _lock = lock::acquire(&paths::lock_path()?, Duration::from_secs(10))?;
        let registry_path = paths::registry_path()?;
        let mut registry_opt = store::load_strict(&registry_path)?;

        // 3. Remove env files (union of ledger + live worktrees).
        let mut env_dirs: std::collections::BTreeSet<PathBuf> =
            ctx.worktree_list()?.into_iter().collect();
        if let Some(registry) = &registry_opt {
            if let Some(project) = registry.find_project(&common_dir_key) {
                env_dirs.extend(project.worktrees.keys().map(PathBuf::from));
            }
        }
        for dir in &env_dirs {
            let env_path = dir.join(".env.portool");
            match fs::remove_file(&env_path) {
                Ok(()) => {}
                Err(err) if err.kind() == std::io::ErrorKind::NotFound => {}
                Err(err) => residue.push(format!("env:{}:{err}", env_path.display())),
            }
        }
        // 4. Remove ledger allocations (only if hooks are neutralized; else a
        //    later checkout re-hooks and re-allocates, so keep them and say so).
        if residue.iter().any(|r| r.starts_with("hook:")) {
            residue.push("allocations-kept:hooks-not-neutralized".into());
        } else if let Some(registry) = &mut registry_opt {
            if registry.projects.remove(&common_dir_key).is_some() {
                store::save(&registry_path, registry)?;
            }
        }
    }

    // 5. Remove the exclude entry.
    if let Err(err) = deinit_exclude(&ctx.common_dir) {
        residue.push(format!("exclude:{err}"));
    }

    // 6. Report — success only when nothing is left behind.
    report_hook_removal(&hook_results);
    if residue.is_empty() {
        println!("portool: deinitialized this project");
        Ok(())
    } else {
        println!(
            "{{\"partial_deinit\":{{\"residue\":{}}}}}",
            serde_json::to_string(&residue).unwrap()
        );
        Err(Error::General(
            "deinit did not fully complete; see partial_deinit above".into(),
        ))
    }
}
```

```rust
/// The interpreter classification of a hook file's first line, from a real
/// tokenization of the shebang rather than a substring test (external review
/// v0.10 P0-6). Only `None`/`PosixShell` are safe to inject a `sh` block into.
#[derive(Debug, PartialEq, Eq)]
enum Shebang {
    None,
    PosixShell,
    Other,
    Unparseable,
}

const SHELL_BASENAMES: &[&str] = &["sh", "bash", "dash"];

fn classify_shebang(content: &str) -> Shebang {
    // A leading UTF-8 BOM means the "#!" is not at byte 0; the kernel won't
    // treat it as a shebang -> we can't reason about it -> fail closed.
    if content.starts_with('\u{feff}') {
        return Shebang::Unparseable;
    }
    let first = match content.split('\n').next() {
        Some(line) => line.strip_suffix('\r').unwrap_or(line),
        None => return Shebang::Unparseable,
    };
    if first.is_empty() && content.is_empty() {
        return Shebang::Unparseable; // empty file
    }
    if !first.starts_with("#!") {
        // No shebang: git executes the hook with `sh`.
        return Shebang::None;
    }
    // Kernels cap the shebang line (~127-256 bytes); an absurdly long one is
    // never a real interpreter path.
    if first.len() > 256 {
        return Shebang::Unparseable;
    }
    let rest = first[2..].trim();
    if rest.is_empty() {
        return Shebang::Unparseable;
    }
    let mut tokens = rest.split_whitespace();
    let interp = tokens.next().unwrap();
    let interp_base = basename(interp);
    // `/usr/bin/env [ -S ] <cmd> ...` -> the real interpreter is the first
    // token that is not `env`, not `-S`, and not another option.
    if interp_base == "env" {
        for tok in tokens {
            if tok == "-S" || tok.starts_with('-') {
                continue;
            }
            return if SHELL_BASENAMES.contains(&basename(tok)) {
                Shebang::PosixShell
            } else {
                Shebang::Other
            };
        }
        return Shebang::Unparseable; // `env` with no command
    }
    if SHELL_BASENAMES.contains(&interp_base) {
        Shebang::PosixShell
    } else {
        Shebang::Other
    }
}

/// The final path component of an interpreter token, stripping any option
/// suffix already handled by the caller.
fn basename(token: &str) -> &str {
    token.rsplit('/').next().unwrap_or(token)
}
```
